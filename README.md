# Restaurant Rating Prediction and Analysis System

Система анализа факторов и прогнозирования рейтинга ресторанов на основе данных Yelp.

## Описание проекта
Проект посвящён исследованию факторов, влияющих на пользовательские оценки ресторанов, и созданию инструмента для прогнозирования рейтинга заведения по его характеристикам.

Работа включает две ключевые части:
- анализ данных и обучение моделей машинного обучения;
- разработку прикладного сервиса для получения прогноза рейтинга.

## Цель проекта
Определить основные факторы, влияющие на восприятие ресторанов пользователями, и построить модель, способную предсказывать рейтинг на основе доступных характеристик.

## Данные
В качестве источника информации использован открытый датасет Yelp:  
https://business.yelp.com/data/resources/open-dataset
Для анализа были выбраны объекты, относящиеся к сфере общественного питания.

## Этапы работы
- первичный анализ и исследование данных;
- предобработка и формирование признаков;
- подготовка выборки для моделирования;
- обучение и сравнение моделей;
- интерпретация полученных результатов;
- разработка REST API;
- создание интерактивного интерфейса на Streamlit.

## Использованные модели
В рамках проекта протестированы следующие алгоритмы:
- Linear Regression
- Random Forest
- Gradient Boosting
Наилучшие результаты показала модель **Random Forest**.

## Результаты
Лучшая модель продемонстрировала следующие показатели качества:
- MAE: 0.503
- RMSE: 0.650
- R²: 0.416
Эти результаты подтверждают, что методы машинного обучения позволяют выявлять значимые закономерности в данных и формировать практически полезные прогнозы, несмотря на субъективность пользовательских оценок.

## Прикладная часть
На основе обученной модели реализованы:
- REST API на FastAPI для предсказания рейтинга;
- веб-интерфейс на Streamlit для ввода характеристик ресторана и получения прогноза.

## Структура проекта
- `app/` — исходный код API и Streamlit-приложения;
- `model/` — сохранённые модели и вспомогательные артефакты;
- `notebooks/` — ноутбуки с анализом и обучением моделей;
- `requirements.txt` — список зависимостей проекта.

## Примечание
Из-за ограничения GitHub на размер файлов обученная модель `model.pkl` может отсутствовать в репозитории. Для локального запуска её нужно поместить в папку `model/`.

## Запуск проекта локально
1. Установить зависимости: pip install -r requirements.txt
2. Поместить файл модели model.pkl в папку model/, если он не загружен в репозиторий.
3. Запустить API: uvicorn app.main:app --reload
4. Запустить Streamlit-приложение: streamlit run app/streamlit_app.py
5. Открыть в браузере:
  API docs: http://127.0.0.1:8000/docs
  Streamlit: http://localhost:8501

## Практическая значимость
Разработанная система может быть полезна для:
- анализа факторов, влияющих на пользовательские оценки;
- прогнозирования рейтинга новых заведений;
- поддержки управленческих решений в ресторанном бизнесе.
